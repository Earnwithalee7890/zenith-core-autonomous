# Zenith AI Design System Rules

## Visual Aesthetics (The "Zenith" Look)
- **Primary Style**: Glassmorphism (semi-transparent backgrounds, background-blur).
- **Color Palette**: Ultra-dark mode (`#020617`). Accents in Neon Sky-400 (`#38bdf8`) and Cyber Purple (`#c084fc`).
- **Glow Effects**: Use `box-shadow` or `drop-shadow` with low opacity to create "Neon Glow" around buttons and cards.
- **Typography**: Inter (Sans-serif) for high readability. Italicized bold headers for a "Speed" feel.
- **Components**: High border-radius (`rounded-[2rem]`) and floating shadows.

## Technical Standards
- **Framework**: Use **React + TypeScript + Tailwind CSS**.
- **Performance**: High-speed rendering. Use CSS transforms instead of layout shifts.
- **Responsiveness**: Perfect scaling on all devices.
- **Animations**: Inject "Floating" animations for icons and "Pulse" effects for active states.

## Zenith UI Library
Always use the `ui_templates` tool to retrieve:
1. `tailwind-config`: For the elite color palette.
2. `dashboard`: For complex data-driven mystery prompts.
3. `zenith-animation`: For that extra "premium" polish.

## Accessibility
- Contrast ratios must be high.
- Use semantic HTML (`nav`, `main`, `footer`, `section`).
    - Navigation bar
    - Hero section with a CTA
    - Features/Benefits grid
    - Contact/Early-access form
    - Footer
